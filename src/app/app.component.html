Dynamicly Loaded routes

This example demonstrates how Angular can load routes dynamically. Uses for this include:

  • Users with different entry points depending on their permissions or settings
  • Pages that load dynamically from a database
  • dynamically updating icons, link text, or components that the links go to without code changes
  • Reducing the amount of code changes, and shifting work to data changes
  • Easier to test and mock for different user's environments
